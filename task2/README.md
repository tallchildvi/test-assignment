# Task 2 ÔÇö Multimodal Animal Verification Pipeline

The animal verification system is a multimodal inference pipeline designed to cross-reference textual descriptions with visual content through the integration of Transformer-based NLP and Deep Learning CV models.

---

## 1. Architectural Design: Multimodal Integration

The `AnimalVerificationPipeline` class encapsulates both models under a single interface, exposing a single `.run(text, image_path) -> bool` method. This deliberate design decision keeps the caller-facing API minimal while hiding the complexity of two entirely different inference stacks operating in parallel.

The pipeline executes three strictly sequential steps:

1. **Entity Extraction (NLP):** The fine-tuned NER model parses input text to identify and normalize animal-related entities into a flat list of lowercase strings.
2. **Visual Classification (CV):** A ResNet18-based classifier identifies the primary animal subject within the provided image and maps the predicted index to an English class label via a hardcoded `idx_to_animal` dictionary ÔÇö a deliberate bridge over the dataset's original Italian folder naming convention (`cane`, `gatto`, etc.).
3. **Boolean Verification:** A single membership check (`image_animal in text_animals`) serves as the final verification gate, returning `True` only if the CV-predicted label is explicitly present among the NER-extracted entities.

---

## 2. Model Architectures

### A. Named Entity Recognition (NER)

The NER module (`ner_train.py`, `ner_inference.py`) fine-tunes `distilbert-base-uncased` for token classification using the HuggingFace `Trainer` API:

| Parameter | Value |
|-----------|-------|
| Base Model | `distilbert-base-uncased` |
| Tagging Scheme | BIO (`B-ANI`, `I-ANI`, `O`) |
| Aggregation | `simple` strategy ÔÇö merges consecutive B/I spans into a single entity |
| Training Data | 3,000 synthetic samples across 14 sentence templates |
| Optimizer | `AdamW` via HuggingFace defaults, LR=2e-5 |
| Batch Size | 16 |
| Epochs | 5 |
| Evaluation | Token-level accuracy on a 15% held-out split |

A critical implementation detail is the `tokenize_and_align` function, which handles the mismatch between word-level BIO labels and DistilBERT's subword tokenization. Only the **first subtoken** of each word receives the gold label; all subsequent subtokens are masked with `-100` and excluded from both loss computation and accuracy evaluation. This prevents the model from being penalized for split-token representations it cannot control.


### B. Image Classification (CV)

The CV module (`cv_train.py`, `cv_inference.py`) implements transfer learning on top of a ResNet18 backbone:

| Parameter | Value |
|-----------|-------|
| Architecture | ResNet18 pre-trained on ImageNet (`ResNet18_Weights.DEFAULT`) |
| Head Modification | Final `fc` layer replaced: `Linear(512 Ôćĺ 10)` |
| Loss Function | Weighted `CrossEntropyLoss` (handles class imbalance) |
| Optimizer | `Adam`, LR=0.001 |
| Batch Size | 32 |
| Data Split | 85% train / 15% validation |

At inference time (`cv_inference.py`), the model is loaded in `eval()` mode with `torch.no_grad()`, and predictions are made via `torch.max(outputs, 1)` over raw logits ÔÇö no softmax is applied, as `argmax` over logits is equivalent for deterministic prediction.

## 3. Data Engineering & Augmentation

### NLP: Synthetic Dataset Generation & Typo Robustness

Since there are no real-world labeled NER datasets for this closed-set task, the training data is **synthetic**, generated by the `generate_dataset()` function. Each sample is constructed by injecting an animal name into one of 14 sentence templates, covering both mid-sentence (`"I can see a {animal} here ."`) and sentence-initial positions (`"{animal} detected in the foreground ."`). The sentence-initial templates are a deliberate design choice to prevent the model from developing a positional bias toward entity detection.

The `apply_typo()` function introduces character-level noise with a **25% probability** per sample using one of three mutation strategies:

| Mutation | Implementation | Example |
|----------|---------------|---------|
| Adjacent swap | `chars[i], chars[i+1] = chars[i+1], chars[i]` | `cat` Ôćĺ `act` |
| Character removal | `chars.pop(idx)` | `dog` Ôćĺ `dg` |
| Vowel substitution | `chars[idx] = random.choice("aeiou")` | `cow` Ôćĺ `caw` |

The typo is applied to the **display token** in the sentence but the BIO label alignment logic still matches against `display_animal.split()`, ensuring correct labeling even for corrupted tokens.

### CV: Class Imbalance & Preprocessing
 
EDA of the Animals-10 dataset revealed significant class imbalance. The mitigation strategy uses inverse-frequency weighting applied directly to the loss function via a custom `calculate_class_weights()` utility (`utils/dataset_distribution.py`). The resulting weights cause the optimizer to penalize misclassifications of underrepresented classes (e.g., elephant, ~1,500 samples) proportionally more than majority classes (e.g., dog, ~5,000 samples), without requiring any resampling or dataset modification.
 
The training transform pipeline applies `RandomHorizontalFlip` and `RandomRotation(15┬░)` **before** `ToTensor()`, ensuring augmentation operates on PIL images rather than tensors ÔÇö the correct order for `torchvision.transforms`. The validation transform intentionally omits these augmentations, applying only resize and normalization to ensure unbiased evaluation.
 
Both transforms use ImageNet normalization statistics (`mean=[0.485, 0.456, 0.406]`, `std=[0.229, 0.224, 0.225]`), which is required for transfer learning from ImageNet-pretrained weights to function correctly.

## 4. Model Evaluation & Testing

### Quantitative Validation

| Metric | NER | CV |
|--------|-----|----|
| Validation Split | 15% | 15% |
| Primary Metric | Token-level accuracy | Val Loss + Val Accuracy |
| Checkpointing | Best accuracy checkpoint | Best val loss checkpoint |
| Evaluation Frequency | Per epoch | Per epoch |

The NER and CV models use different primary checkpointing criteria by design: NER optimizes for **accuracy** (appropriate for a token classification task with heavily imbalanced `-100` masked tokens), while CV checkpoints on **val loss** (more sensitive to early divergence in a regression-style cross-entropy landscape).

### Feature Dimensionality Analysis (EDA)

Image dimension analysis conducted during EDA informed the choice of the `224├Ś224` resize target:

| Metric | Value |
|--------|-------|
| Average Width | 278 px |
| Average Height | 241 px |
| Min Dimensions | 153├Ś101 px |
| Max Dimensions | 300├Ś300 px |

Since the mean resolution (278├Ś241) is close to and mostly above the 224├Ś224 target, the resize operation **downsamples** the majority of images rather than upsampling them. Downsampling preserves structural information without introducing interpolation artifacts, making `224├Ś224` an efficient and artifact-free choice for this dataset.
